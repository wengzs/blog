# 小型团队管理之激励篇

观点：激动人心的事，才能激励人心。

## 关于成长

通过一次面对面、一对一的深度谈话，了解团队成员的最真实的意向。可以提前先预设好一些问题，比如：

1. 自我定位相关问题：
   1. 你认为前端是一个什么角色？
   2. 认为自己这个角色对小组、公司来说是否重要？
2. 学习规划相关问题：
   1. 在未来一个月希望掌握什么技能？
   2. 在未来三个月希望掌握什么技能？
   3. 在未来一年希望掌握什么技能？
3. ...

通过这些问题，逐步的挖出团队成员的内心意向，以技能成长规划为例，按照下面的思路进行引导：

1. 按照内心意向，结合现有业务，给出比较实贴合实际的技能成长方向。
2. 给出需要必要的资料，如文档手册、项目资料等。
3. 给出相关 OKR，按时间节点拆分好 KR，定期 review 学习成果。
4. 给出可落地的方向。作为管理者，需要尽可能的创造条件给他们在项目中实践新学会的技能。
5. 协助他们优化自己的学习路径，并整理输出相关资料，反哺团队。
   1. 看什么文档、以什么顺序、怎么搭环境、demo0、demo1、demo2...踩了什么坑...
   2. 输出一些关键性知识的考题
6. 尽可能的对内/对外曝光他们的心得，以提升他们的个人知名度（帮他们刷声望）。

我们暂且把`看点文档 => 写点博客 demo => 看点其他文档 => 写点博客 demo`这种学习模式称为`自闭型学习模式`，既不注重团队业务方向，又不注重实践，还远离团队/社区交流。

正确的做法，应该是`开放性学习模式`：`学习符合团队业务方向的技能 => 在具体业务中应用 => 总结学习路径 => 总结知识点(考题) => 反哺团队 & 对内/对外曝光交流`。一举多得，且具有很好的激励性。

## 关于业务

一个好的管理者，要为团队创造各种成长练兵机会。

1. 跟上时代：小步快跑的推广新的技术。
2. 开放权限：在非关键节点上，尽量只管`要做什么`，少管`怎么做`。
3. 寻找新的业务增长点：有增长的业务、在扩张中的团队机会最多。

## 关于氛围

我比较喜欢《OKR 工作法》中的每周五的庆祝会。团队辛苦了一整个星期，终于到了版本发布的一天，应该好好庆祝一下。
当然大部分人还是觉得“庆不庆祝没什么”、“事情做完了就好了”。尝试去打破这个观点，不要错过任何一个在正式场合肯定、鼓舞团队的机会。我参加过好几次在例会上进行那种不怎么“高级”的月度表彰会，都是很普通的 ppt、主持人，以及有点尬场的获奖感言。但是最后大家都是笑着离开会场，还讨论着怎么处理到手的奖品，真香无处不在。很多时候，在尴尬和有趣之间，只差一个打趣。

尝试着坚持这种庆祝，告诉大家，`我们取得了很不错的成果`。
