# 招聘面试需要注意的一些细节

最近一段时间，大约面了50+位中高级前端位候选人，在此总结了一些作为面试官需要注意的细节。

# 预习

提前1～2天熟悉下候选人的简历，了解候选人的技术特点、对岗位的预期值。提前标注好面试的重点。如果候选人的资历、能力等明显超过了面试官，可以提前准备好面试题。

# 互相介绍

开场白千篇一律，但必不可少。除了润滑面试氛围，最重要还是能够建立起候选人和面试官之间的初步信任。信任感往往是除了薪水、平台之外的决定性因素。技术面终面面试官，往往就是候选人最终的团队leader，一个好的开头，有助于今后的团队管理。

对于候选人的自我介绍，需要关注：实际的工作年限、在最近一两份工作中所承担的角色，为何离职。 面试官也需要做个自我介绍，这个介绍放到技术面结束之后，在提问环节之前。 面试官需要讲清楚：我们是谁，我们在做什么，我们 / 我们部门需要什么样的人，能够给这个岗位的员工什么样的成长机会、会遇到什么样的挑战。遇到合适的候选人，可以多花点时间讲讲这方面的东西。一线员工最了解一线岗位的情况，等到了hr面或者说CTO/CEO面，候选人获得的信息会比较抽象，不容易认识到"我来这儿之后具体要做什么"这个问题。

# 考察能力

对着题库读题不可取。面试的体验太差，同时给候选人的一种"面试官有点菜"的感觉。最好是针对简历提问。可以增加这两点考察：

  1. 考察下限 挖细节。细节决定下限。
  2. 考察上限 设计一些开放性问题，主要问思路。

# 时间

控制面试时间在 30-60 分钟左右。如果确定不满足预期，马上停止面试。

# 同理心

对于一些开放性问题 / 比较难的问题，需要面试官逐步引导候选人的思路，不要一言不发搞压力面。候选人答错了或者没思路，可以安慰下，跳过这个问题。技术面还是主要考察候选人的优点，当候选人的能力能满足hc的要求，稳定性考察这种就交给hr好了。不要在面试中评价候选人的能力、资历等。不要当面给出结论。毕竟我们也不喜欢别人对我们的能力指指点点，尤其是这种正式场合。

# 平常心

候选人的能力、薪资、薪资要求往往会超出面试官对这个岗位的认知。比如要价30～40的候选人面20～30的岗，会对面试官造成一些压力；比如招进来的人拿的钱比你多；比如候选人最终的职级定级比你这一面定的高，等等。此时需要保持好平常心，专注于面试。
